## Sobre o projeto e objetivo

### Overview

Este projeto possuí finalidade de estudo e treinamento, onde o objetivo maior é colocar em prática Clean Arctecture - 
que, quando iniciado o projeto, estava lendo o livro "Clean Arctecture, A Craftsman's Guide to Software Structure and 
Design" de Robert C. Martin e demais - para entender os conceitos e onde eles se aplicam.

Encare este projeto mais como um "desafio de programação", só que de "um nível mais alto". Algo como escrever tentar 
escrever "Hello World" em um microondas... Não é coisa que se vê todo dia e nem é a prática mais ortodoxa, mas quem 
faz aprende uma coisa ou outra! É básicamente isso. 

**Em resumo**: um projeto de aprendizagem multidiscliplinar, sem muito compromisso com a entrega, apenas com o
processo e a aprendizagem.

### Objetivos do projeto

**Primários:**

1. Entender melhor Clean Arctecture e seus contextos de aplicação
2. Explorar e descobrir conceitos atrelados não abordados no livro de Robert C. Martin
3. Aplicar Clean Code e SOLID
4. Compreender melhor Entities
5. Entender as limitaçoes da Clean Arctecture, e quais conceitos a expandem
6. Treinar Python em Clean Arctecture
7. Treinar desacoplamento

**Secundários:**
1. Entender mais sobre Policies e Service Domain
2. Aprender mais e arquiteturar melhor Use Cases
3. Aplicar Logging no contexto de arquitetura Clean
4. Aplicar e compreender melhor inversão de dependência
5. Entender melhor VO e colocar em prática

**Terciários:**
1. Criar familiaridade com VS Code e descobrir extensions uteis para Python
2. Aprender mais sobre Abstração em Python (e biblioteca ABS e Typing)
3. Treinar diferentes formas de Mains

**Objetivos extras:**
1. Aprender sobre BOT Whatsap, Telegram e/ou Discord

### Atualmente

Atualmente o projeto encontra-se parado, tanto por falta de tempo quanto por já ter dado excelentes frutos.
Aprendi muito com ele, mas não sei se irei termina-lo em si, até por conta da lingaugem primária ser Python
e haverem muitas discussões sobre a linguagem não sofrer dos problemas que a arquitetura tenta sanar.

Talvez não parece, já que o projeto e o estado atual do código não refletem o processo de pesquisa, mas 
aprendi aqui muito mais do que vim procurar.

De todo modo, é um projeto simples e bom; *quem saiba eu volte aqui algum dia e refatore ele para outra
linguagem...*

## Sobre o uso do código

Todo o código aqui foi feito por mim para testar minhas habilidades. Se inspire e use a vontade, porém
com a ética de não tomar para si. Não pretendo qualquer tipo ou forma de envolvimento com direitos de
qualquer modo, e espero o mesmo de vocês; o código está aqui a exposição para uso público e como backup
pessoal do projeto em si.

## Roadmap

O Road Map quando iniciei o projeto não foi muito bem definido, fui fazendo de acordo com o que me dava
mais dúvidas no momento, sendo guiado por este fluxo de aprendizagem. Porém, seria algo como:

0.1 Camada Dominio pronto e funcional, com Entities, VOs, Policies e afins.

0.5 Camada Aplication funcional e pronta para dar apoio as extenções futuras

0.6 Principais cálculos prontos (adição, subtração, multiplicação e divisão)

0.8 Versão básica funcional em CLI

1.0 Versão funcional em CLI com permanencia básica

1.4 Autotests prontos e interace gráfica (pyautogui ou TKInter) pronta

1.6 Funções mais complexas prontas (Exponenciação, Fatoração...)

1.7 Função de cálculo em lote 100% funcional

1.8 Telegram BOT funcional

1.9 Whatsap BOT funcional

2.0 Discord BOT Funcional

2.4 BD para salvar sesões de cálculo, hisstórico e preferências por usuário
